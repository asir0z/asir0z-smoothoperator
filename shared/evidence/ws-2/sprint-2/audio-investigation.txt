=== ROOT CAUSE ===

VirtualBox VM audio disabled at hypervisor level (audio driver null, output off).
Guest PipeWire stack is healthy; playback fails because host routes no audio device.

Not a PipeWire/WirePlumber misconfiguration.
Not a Hyprland or dotfiles issue.

=== FIX (smallest reproducible) ===

1. Power off VM
2. Windows: bash linux/install/enable-vm-audio.ps1
   (or: VBoxManage modifyvm Arch-Engineering-Workstation --audio-driver default --audio-out on)
3. Start VM · log in to Hyprland
4. Verify: wpctl status + pw-play test tone

Repository reproducibility:
  linux/install/create-vm.ps1 updated: --audio-driver default --audio-out on (new VMs)
  linux/install/enable-vm-audio.ps1 added (existing VM one-shot)

No shared dotfiles change required.
